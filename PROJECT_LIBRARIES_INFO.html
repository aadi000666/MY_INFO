Project Technical Report

Complete breakdown of Packages, Libraries & Architecture.

NPM Packages (Dependencies)

Yeh woh packages hain jo `package.json` me install hain aur project ki backbone hain:

react v18.2.0 Core Package

The primary UI library for building the application.

Kyu? (Reason)

Aapke portfolio ke components (Hero, Navbar, etc.) ko dynamic aur interactive banane ke liye React ka use kiya gaya hai. Yeh JSX use karta hai jo coding ko easy banata hai.

react-dom v18.2.0 Renderer

Entry point for web-based rendering.

Kyu? (Reason)

Yeh package ensure karta hai ki React components actually Chrome/Safari/Edge browser me display ho sakein.

Dev-Packages (Development Only)
vite v5.0.0 Build System

Modern build tool for lightning-fast development.

Kyu? (Reason)

Development ke time "Hot Module Replacement" (HMR) handle karne ke liye. Jab aap code change karte hain, toh bina refresh kiye change show ho jaata hai.

@vitejs/plugin-react v4.0.0 Plugin

Connectivity bridge between Vite and React.

Kyu? (Reason)

Vite ko React ke advanced features (jaise Fast Refresh) samjhane ke liye is package ki zarurat padti hai.

@types/react Type Support

TypeScript type definitions (for Autocomplete).

Kyu? (Reason)

Code likhte waqt VS Code me "suggestions" aur "autocomplete" sahi se aaye, isliye yeh development support package use hota hai.

External Icons & Assets

Maine project check kiya hai, aap kisi external "Package" (jaise react-icons) ka use nahi kar rahe icons ke liye. Saare icons **Inline SVG** format me hain.

Fayda (Benefit)

Isse aapka project bohot light-weight rehta hai kyunki koi extra dependency load nahi hoti.
