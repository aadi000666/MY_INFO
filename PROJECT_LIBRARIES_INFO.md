## 📦 NPM Packages (Dependencies)
These are installed via `npm` and are located in your `node_modules` folder:

### 1. react (v18.2.0)
- **Role:** The core framework.
- **Why?** Components like `Hero`, `Navbar`, and `Projects` are built using React to make them interactive and efficient.

### 2. react-dom (v18.2.0)
- **Role:** Web Renderer.
- **Why?** This package connects React's logic to the browser window so your website can be seen.

## 🛠️ Development Packages (DevDependencies)
These are used only during coding and building, not on the live site:

### 3. vite (v5.0.0)
- **Role:** Next Generation Frontend Tooling.
- **Why?** It provides an extremely fast development server with Hot Module Replacement (HMR).

### 4. @vitejs/plugin-react (v4.0.0)
- **Role:** Vite React Connection.
- **Why?** Essential for Vite to handle React-specific syntax (JSX) and features.

### 5. @types/react
- **Role:** Type Definitions.
- **Why?** Provides coding suggestions (IntelliSense) in VS Code while writing React code.

## 🎨 Asset & UI Architecture
**Important Note:** Some features look like they use packages but are actually **Custom Built**:

- **Icons:** No `react-icons` package used. All icons are optimized **Inline SVGs**.
- **Animations:** No `framer-motion` or `gsap` used. All animations are pure **CSS3 Keyframes**.
- **Particles:** No `tsparticles` used. Built using native **HTML5 Canvas API**.
